# 手机扫码批量上传发票 — 设计文档

**日期：** 2026-04-12  
**状态：** 待实现

---

## 背景

用户收集纸质发票时需要拍照，目前流程是：手机拍照 → 传到电脑 → 手动导入 App。步骤繁琐。

目标：手机拍完照直接扫码上传，图片自动进入识别队列，无需任何中转步骤。

---

## 范围

只做手机扫码批量上传功能，不涉及邮箱、文件夹监控等其他入口。

---

## 架构

新功能作为新的"发票入口"，最终汇入现有 `_start_ocr()` 流程，识别/确认/导出逻辑不变。

```
手机扫码上传  ──▶  临时目录  ──▶  _start_ocr()  ──▶  识别  ──▶  确认  ──▶  导出
```

---

## 组件设计

### 1. `core/phone_server.py` — 本地 HTTP 服务

- 启动时监听局域网 IP，端口 8765（可配置）
- 只绑定局域网接口，不对公网暴露
- 提供两个接口：
  - `GET /` — 返回手机上传页面（纯 HTML + JS，内嵌，无需外部依赖）
  - `POST /upload` — 接收文件，保存到临时目录，通过回调通知主线程
- 上传页面功能：
  - 调用系统原生文件选择器（`<input multiple accept="image/*,.pdf">`）
  - 支持多选，显示已选文件列表和数量
  - 上传进度显示（逐张上传，实时反馈）
  - 上传完成后显示"已上传 N 张，可继续选择"
- 文件校验：只接受 jpg/jpeg/png/pdf，单文件限制 50MB，拒绝其他类型
- 使用 Python 标准库 `http.server`，在独立线程中运行，不引入新依赖
- 通过 `threading.Event` 控制启停

### 2. `ui/phone_upload_dialog.py` — 上传对话框

- 点击侧边栏"手机上传"后弹出
- 内容：
  - 二维码图片（显示局域网访问地址）
  - 文字提示："同一 WiFi 下，用手机扫码选择发票照片上传"
  - 实时状态：已上传 N 张 / 正在识别中
  - "完成"按钮关闭对话框并停止服务
- 二维码使用 `qrcode` 库生成，渲染为 QPixmap 显示
- 对话框关闭时自动停止 HTTP 服务，清理临时目录

### 3. `ui/sidebar.py` — 新增入口

- 侧边栏新增"手机上传"菜单项，图标用 `qtawesome` 现有图标库

### 4. `ui/main_window.py` — 集成

- `_on_nav()` 新增 `phone_upload` 分支，打开对话框
- 对话框通过信号将上传完成的文件路径列表传回，调用 `_start_ocr()`

---

## 数据流

```
手机选图
  └▶ POST /upload (multipart/form-data)
       └▶ 保存到 ~/.invoice_assistant/phone_upload_tmp/
            └▶ 回调信号 → MainWindow._start_ocr([paths])
                 └▶ OCRWorker 识别
                      └▶ 结果写入 SQLite
                           └▶ UI 刷新
```

## 错误处理

| 场景 | 处理方式 |
|------|---------|
| 端口 8765 被占用 | 自动尝试 8766、8767，仍失败则提示用户 |
| 手机与电脑不在同一 WiFi | 二维码下方提示"请确保手机和电脑在同一网络" |
| 上传文件格式不符 | 服务端返回 400，手机页面显示"不支持该文件格式" |
| 上传文件超过 50MB | 服务端返回 413，手机页面显示"文件过大" |
| 网络中断 | 手机页面显示上传失败，可重试 |

## 不在范围内

- 手机 App（不需要，用浏览器网页即可）
- 跨网络传输（不支持，只限局域网）
- 邮箱自动抓取
- 文件夹监控
